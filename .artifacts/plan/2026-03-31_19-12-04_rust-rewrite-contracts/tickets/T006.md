# T006: Rebuild the tool execution and agent event contract

**Summary**: Restore concurrent tool execution, tool-result message generation, and agent state/event ordering so the Rust rewrite does not change how tool calls, steering, follow-ups, and terminal errors appear to Python consumers.

**Owner**: runtime

**Estimate**: 10h

**Dependencies**: T001,T005

**Target milestone**: M3

**Acceptance test**: `uv run pytest tests/test_parallel_tool_execution.py tests/test_agent.py tests/test_usage_contracts.py -q`

**Files/modules touched**:
- tinyagent/agent_tool_execution.py
- tinyagent/agent_loop.py
- tinyagent/agent.py
- rust/src/lib.rs
- rust/src/types.rs
- tests/test_parallel_tool_execution.py
- tests/test_agent.py
- tests/test_usage_contracts.py

**Steps**:
1. Preserve concurrent tool execution and the current ordering guarantees for `ToolExecutionStartEvent`, `ToolExecutionUpdateEvent`, `ToolExecutionEndEvent`, `MessageStartEvent`, and `MessageEndEvent`.
2. Keep `ToolResultMessage` generation and error-text fallbacks aligned with the current Python execution path.
3. Ensure steering, follow-up, and terminal-error behavior still drive the same `AgentState` updates and public event stream ordering.
