# T005: Rebuild the loop-to-provider and proxy streaming contracts

**Summary**: Restore the boundary behavior between `AgentContext` and provider `Context`, and preserve the proxy event protocol including partial-message mutation, stop-reason normalization, and incremental tool-call JSON accumulation.

**Owner**: runtime

**Estimate**: 10h

**Dependencies**: T001,T003,T004

**Target milestone**: M3

**Acceptance test**: `uv run pytest tests/test_contracts.py tests/test_tool_call_types_harness.py -q`

**Files/modules touched**:
- tinyagent/agent_loop.py
- tinyagent/proxy.py
- tinyagent/proxy_event_handlers.py
- rust/src/lib.rs
- rust/src/types.rs
- tests/test_contracts.py
- tests/test_tool_call_types_harness.py

**Steps**:
1. Preserve `_build_llm_context(...)` behavior so `transform_context` and `convert_to_llm` still define the only conversion boundary from `AgentContext` to provider `Context`.
2. Keep assistant stream events and final assistant messages consistent with current start/update/end handling in the loop.
3. Re-implement proxy event parsing so text, thinking, and tool-call events mutate partial assistant messages exactly as the current handlers do, including `partial_json` accumulation and terminal `done` or `error` semantics.
