# T004: Rebuild the typed Rust binding provider contract

**Summary**: Restore `tinyagent.rust_binding_provider` as the stricter typed surface for the in-repo binding, including `RustBindingModel`, provider defaults, allowed APIs, explicit payload models, and `exclude_none` serialization behavior.

**Owner**: runtime

**Estimate**: 8h

**Dependencies**: T001,T002

**Target milestone**: M2

**Acceptance test**: `uv run pytest tests/test_rust_binding_provider.py -q`

**Files/modules touched**:
- tinyagent/rust_binding_provider.py
- rust/src/lib.rs
- rust/src/types.rs
- tests/test_rust_binding_provider.py
- docs/api/providers.md

**Steps**:
1. Preserve `RustBindingModel` validation rules, including accepted `api` values, provider default base URLs, and env-var API-key fallback.
2. Keep `BindingModelPayload`, `BindingContextPayload`, and `BindingOptionsPayload` aligned with the exact dicts expected by the native binding entrypoint.
3. Ensure typed-provider serialization still rejects legacy aliases and non-`model_dump` messages instead of silently coercing them.
