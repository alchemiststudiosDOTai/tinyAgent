**Summary**: Add one shared baseline for the Rust rewrite so provider payloads, assistant messages, events, stop reasons, and usage shapes are asserted from the current Python contract before deeper Rust changes begin.

**Owner**: runtime

**Estimate**: 4h

**Dependencies**: none

**Target milestone**: M1

**Acceptance test**: `uv run pytest tests/test_alchemy_provider.py tests/test_rust_binding_provider.py tests/test_usage_contracts.py tests/test_contracts.py -q`

**Files/modules touched**:
- tests/test_alchemy_provider.py
- tests/test_rust_binding_provider.py

- docs/contract-analysis.md

**Steps**:
1. Enumerate the current contract cases already enforced by the compatibility provider, typed provider, usage tests, and proxy/tool contract tests.
2. Add or tighten assertions so the same cases pin exact field names, optionality, import/load behavior, and stop-reason/event literals.
3. Update `docs/contract-analysis.md` so the checked cases match the test matrix and no contract item is left as an untracked note.
